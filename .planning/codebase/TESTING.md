# Testing & Verification

## Testing Strategy
HydraWatch uses **evaluation scripts** against published academic baselines rather than traditional unit test suites.

## Model Evaluation

### Training Metrics
- Generated by `scripts/train.py`, saved to `outputs/`:
  - `xgboost_metrics.json` — F1, precision, recall, AP, confusion matrix
  - `lstm_metrics.json` — val loss, anomaly threshold, detection F1/P/R
  - `gat_metrics.json` — best F1, localisation accuracy

### BattLeDIM Ground Truth Evaluation
- Script: `scripts/evaluate.py`
- Evaluates against `2019_Leakages.csv` (published ground truth)
- Metrics computed:
  - **True Positive:** alert fired within 30 min of actual leak start
  - **False Positive:** alert with no corresponding ground truth leak
  - **Detection Delay:** median minutes between leak start and first alert
  - **Localisation Accuracy:** % of alerts where suspect_nodes contains true leak node
- Published baseline to beat: **MNF detector F1 = 0.50, detection delay ~6 hours**

### Runtime Evaluation Endpoint
- `GET /metrics` — returns live F1, precision, recall, detection delays during replay
- Tracked in `_state["metrics"]` dict during WebSocket replay sessions

## Interactive Verification

### LeakDB Scenario Replay
- **WebSocket:** `ws://localhost:8000/ws/stream`
- Replays any of 500 LeakDB scenarios at configurable speed
- Dashboard shows real-time model output, network graph, alerts, SHAP
- Visual verification: node colors change blue→amber→red as leak develops

### BattLeDIM Live Replay
- **Endpoint:** `GET /replay/start`
- Streams full 2019 SCADA year through the ensemble
- Ground truth comparison happens automatically
- Results available at `GET /metrics`

## Build Verification
- **Frontend:** `cd frontend && npx vite build` — verifies all imports resolve, JSX compiles
- **Backend:** `python -c "from backend.main import app"` — verifies all model/data imports
- **Models:** `python -c "from backend.models.ensemble import EnsembleDetector"` — verifies model availability

## No Formal Test Suite
- No `pytest`, `jest`, or CI/CD configuration
- Quality assurance is via:
  1. Training metrics vs baselines
  2. Interactive replay with visual dashboard inspection
  3. BattLeDIM ground truth evaluation script
  4. Build verification (Vite build + module imports)

## Data Integrity Checks
- `loader.py` validates scenario directory structure before loading
- `features.py` handles edge cases (rolling windows at t=0, empty demand data)
- `ensemble.py` gracefully degrades when individual models are unavailable (passes check if model is None)
